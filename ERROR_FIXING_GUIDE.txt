COMMON ERRORS & SOLUTIONS
==========================

ERROR 1: "Authentication Failed"
---------------------------------
Cause: Wrong email/password or account doesn't have app-specific passwords

Solution:
Gmail:
  1. Go: https://myaccount.google.com/apppasswords
  2. Select "Mail" → "Windows Computer" (or your OS)
  3. Copy the 16-character password (NO SPACES)
  4. Paste into the app (this is NOT your regular password)

Zoho:
  1. Go: https://mail.zoho.com
  2. Settings → Security
  3. Generate app-specific password
  4. Use that password, not your account password

Outlook/365:
  1. Go: https://account.microsoft.com/security
  2. Advanced security options
  3. Create app password
  4. Use the generated password


ERROR 2: "SMTP Connection Timeout"
-----------------------------------
Cause: Firewall blocking port 587 or network latency

Solutions:
  - Check firewall allows outbound port 587
  - Try from different network (mobile hotspot for test)
  - Increase timeout in code (already done in fixed version)
  - Try different email provider


ERROR 3: "SMTPRecipientsRefused - 550 Invalid recipient"
---------------------------------------------------------
Cause: One of your email addresses in CSV is invalid

Solution:
  - Check CSV file for typos in email column
  - Remove rows with invalid emails
  - Test with valid test emails first (use your own email)


ERROR 4: "Module not found: openpyxl"
--------------------------------------
Cause: Excel support library not installed

Solution:
  pip install openpyxl


ERROR 5: "Address already in use"
----------------------------------
Cause: Flask server already running on port 5000

Solutions:
  - Close the previous window with server running
  - Or: Change port in app.py line 445: app.run(port=5001)
  - Or: Kill the process: lsof -i :5000 (macOS/Linux)

QUICK FIX STEPS
===============

If emails aren't sending:

1. Check the campaign log file
   ls *.log (or dir *.log on Windows)

2. Look for VERIFY failures
   If VERIFY fails → fix password in your sender account

3. Look for EMAIL failures
   If emails fail → check recipient email addresses in CSV

4. Check Flask console output
   Error messages appear in terminal too

5. Increase verbosity
   - Add print() statements after failed sends
   - Or check the .log file (now saved automatically)


---

ADVANCED DEBUGGING
==================

Enable debug mode to see SMTP conversations:

In app.py, find "server = smtplib.SMTP" and add:

    server = smtplib.SMTP(config['smtp_server'], config['smtp_port'], timeout=20)
    server.set_debuglevel(1)  # Add this line
    server.starttls()
    ...

This will print full SMTP conversation to console.

NEED MORE HELP?
===============

Check these files:
- campaign_*.log ← Detailed email sending logs
- Your email provider's security logs (Gmail → Security Hub)
- Windows Firewall settings (port 587 must be open)
- Antivirus might block SMTP (whitelist localhost:587)

Test from Command Line:
telnet smtp.gmail.com 587

If it connects, port is open.
If it times out, firewall blocking.
